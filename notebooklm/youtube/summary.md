# Шаблон структурированного summary для Obsidian

> [!abstract] Делай summary в Markdown по фиксированным блокам: контекст → H1 → abstract-callout → разделы с буллетами → теги → источники. Пиши применимо и конкретно, сохраняя причинно-следственную логику исходника и не добавляя новых фактов.

---

## Старт заметки

• Начни с 1–2 строк контекста: что за заметка и откуда материал (без воды).
• Затем поставь H1 с названием заметки: `# <Название заметки>`.
• Сразу после H1 добавь callout-резюме: `[!abstract]` + главная концепция в 1–3 предложениях.
• После callout обязательно вставь разделитель `---`.

## Разделы

• Создавай столько разделов, сколько нужно (N) — количество заранее не фиксируется.
• Каждый раздел оформляй как `## <Короткое название раздела>`.
• Внутри раздела — буллеты `•`, количество пунктов произвольное (M).
• Каждый пункт — одна строка: действие/правило/наблюдение, максимально применимо.

## Качество пунктов

• Первый пункт в разделе делай “операционным” (что делать / какое правило).
• Следующие пункты используй для применения: как делать, пример, уточнение.
• Добавляй дополнительные пункты только по необходимости: ограничения, условия, тайминг, лучший вариант, альтернативы.
• Не оставляй пустых пунктов и не добавляй “для красоты”.

## Смысл и переписывание

• Не копируй формулировки исходника: перефразируй.
• Сохраняй причинно-следственные связи: “сделай X, потому что Y”.
• Не добавляй фактов, которых нет в исходном тексте (никаких домыслов/расширений).
• Держи фокус на конкретике и применимости, а не на пересказе.

## Завершение заметки

• После последнего раздела поставь разделитель `---`.
• В конце добавь строку тегов: 3–8 штук, по смыслу (формат `#tag1 #tag2 ...`).
• Затем добавь блок “Ссылки и источники” списком `- ...` (3 пункта или сколько реально есть).
