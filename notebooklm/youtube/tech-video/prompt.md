ЗАДАЧА
Составь конспект технического видео по предоставленной ссылке и транскрипту (транскрипт доступен тебе в этой сессии). Output должен быть Markdown-файл с YAML front matter и телом строго по шаблону пользователя.

ОГРАНИЧЕНИЯ (анти-галлюцинации)
1) НЕЛЬЗЯ добавлять знания извне видео. Если факт/широкое утверждение не звучит в транскрипте — не пиши его. Вместо этого: "в видео не сказано" или пропусти.
2) КАЖДЫЙ ключевой тезис в Summary/Выводах/Best practices должен иметь опору: (а) ссылка на цитату Q# или (б) на дословный фрагмент кода/команд из раздела "Команды, код, конфиги".
3) Цитаты должны быть дословные. Старайся держать одну цитату ≤ 25 слов (если возможно без потери смысла).
4) Таймкоды, транскрипт и субтитры НЕ хранить. Никаких скриншотов.
5) Язык конспекта RU. Термины/команды можно оставлять EN. Не переводить команды.

ЦЕЛЕВОЕ ПОКРЫТИЕ (полнота)
- Допустимы пропуски до ~20% деталей, но нельзя пропускать:
  a) определения, которые автор объясняет,
  b) команды/код/конфигурации, которые автор показывает,
  c) best practices / анти-паттерны, которые автор проговаривает,
  d) финальные выводы автора (если есть),
  e) ссылки/ресурсы, которые автор упоминает.

ВЫХОДНОЙ ФОРМАТ
- Сгенерируй один Markdown-документ.
- YAML поля: source, url, video_id, title, channel, published_at (если можешь), watched_at (сегодняшняя дата), duration_min (если можешь), type, tags, topics (опционально), status=draft, usefulness (не заполняй — оставь пустым).
- Затем разделы по шаблону (Summary → Темы и раскрытие → Определения → Код → Best practices → Ссылки → Цитаты → Выводы → Шаги → Вопросы).

САМОПРОВЕРКА ПЕРЕД ВЫВОДОМ
- Составь список всех утверждений (claims) из Summary/Best practices/Выводов.
- Для каждого claim укажи Q# или код-опору.
- Если claim без опоры — удали его или преврати в вопрос/неопределённость.
